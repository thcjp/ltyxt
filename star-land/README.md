## 这个项目的故事

小甜是一个二年级的女孩儿。她的爸爸老田每天忙于工作，早出晚归，渐渐疏忽了对小甜学习的管理。他总觉得"孩子还小，慢慢来"，直到那天——班主任打来电话，请家长到学校谈话。

老田坐在教室里，看着小甜的成绩单，心里像被针扎了一样：孩子的成绩已经滑到了班级倒数。

回家的路上，小甜低着头不说话。老田想发火，却发不出——因为他知道，真正该被责怪的是自己。他太放手了，以为孩子能自己管好自己，却忘了八岁的孩子根本没有这个自控力。

那天晚上，老田做了一个决定：他要亲自陪小甜把落下的功课补回来。可当他翻开课本，发现自己也懵了——凑十法是什么？CPA教学法怎么用？英语单词的发音他也不确定，拼音的声调他也搞不清。

于是，老田想到了用AI快速完成整个教学体系。他把自己对女儿的愧疚、对教育的理解、以及从全球教学法中汲取的精华，全部倾注进了这个程序。他给它起了一个名字：**老田和小甜甜的游戏屋**。

这个程序有一个刻意的设计：**它必须由父母陪着孩子一起用**。老田太清楚，小甜之前的学习之所以滑坡，根源就在于他太放手了。孩子太小，自控力是一点一点培养起来的，不能指望她独自面对屏幕就能学好。所以游戏屋里的每一节课，都有"I Do（家长示范）→ We Do（亲子合练）→ You Do（孩子自练）"三个环节，家长是不可或缺的角色。

老田把他对小甜所有的亏欠，都变成了代码里的一行行温暖：答对题了，程序会喊小甜的名字说"小甜你真棒"；连续学习有星星奖励，攒够了可以兑换去公园玩；每天结束时，父女俩一起记录"今日高光时刻"。

这不是一个冷冰冰的教育App，而是一个父亲写给女儿的道歉信，也是一份承诺：**从今天起，爸爸陪你一起学。**

## 项目定位

**老田和小甜甜的游戏屋** 是一套面向小学 1-6 年级（语文、数学、英语三科）的**家庭亲子共学补救学习系统**，兼具商业产品价值与教育公益属性。

### 产品定位

- **目标用户**：小学低年级学生及其家长，尤其是孩子学习出现滑坡、家长想辅导却不知从何下手的家庭
- **核心场景**：家长下班后陪孩子用 30 分钟完成一节"诊断→示范→合练→自练"的完整补救课时
- **差异化优势**：
  - 不是"扔给孩子自己学"的题库 App，而是**强制家长参与**的共学系统
  - 不是按年级从头教，而是**诊断先行、精准补救**，从孩子不会的第一个地方开始
  - 不是冷冰冰的练习题，而是**14 种 GSAP 教学动画 + 智能语音**的多感官体验
  - 不是必须联网的在线课程，而是**完全离线运行**的单文件应用，任何电脑双击即用

### 商业价值

| 维度 | 说明 |
|------|------|
| **内容壁垒** | 18 个课程包、144 个单元、720 课时、5040 道练习题，三科完全对等且对标最新课标 |
| **技术壁垒** | 智能动画推断系统、TTS 拼音/表情清洗、库存经济系统、单文件离线打包 |
| **可扩展性** | 课程数据为结构化 TypeScript 文件，AI 可按模板快速扩展年级与科目 |
| **跨平台** | 一套代码，Web / Windows 桌面 / Android 三端部署 |
| **儿童心理学** | 双货币激励、库存稀缺感、成就解锁、个性化表扬，正向行为强化 |

### 儿童心理学考量

游戏屋在设计中深度融入了儿童心理学原则：

- **奖励系统**：星星（日常）+ 钻石（成就）双货币体系，配合**库存稀缺感**（心愿商店库存有限、售罄置灰），让孩子学会"攒星星—做选择—延迟满足"
- **学习体验**：I Do → We Do → You Do 三段式流程提供安全感支架，正确率 ≥80% 才过关避免挫败感
- **个性化反馈**：可设置孩子名字，答对题时 TTS 朗读"小甜你真棒"，让孩子感受到被看见
- **多感官刺激**：视觉（动画+emoji）+ 听觉（TTS 语音）+ 动觉（拖拽题目），符合蒙台梭利多感官学习法

## 项目最大特点

### 注意
本项目完全是AI完成，我在和小甜一起学习的过程中发现有些练习还是有些问题，特别是填空题，答案和应试教育一样，是标准答案。为此我专门开发了**家长批改**功能：在填空题和闪卡题中，当系统判定孩子答错时，会出现"家长批改"按钮，家长可以查看孩子的答案和参考答案，主观判断是否算孩子答对。这既解决了标准答案过于死板的问题，也逼迫家长必须陪着孩子一起学习——因为只有家长才能判断孩子的答案是否合理。

### 一、学习方法全球最先进

本课程深度融合全球十大教学方法论，并非简单堆砌，而是根据补救教学场景做了系统性整合：

| 教学法 | 来源 | 在游戏屋中的应用 |
|--------|------|-----------------|
| RTI分层干预模型 | 美国特殊教育 | 阶段零精准诊断→分层干预→逐级加码 |
| 显性直接教学（EDI） | 美国直接教学法 | 每课时严格 I Do→We Do→You Do 三步流程 |
| 支架式教学 | 维果茨基最近发展区 | 全支架→半支架→微支架，逐步撤除 |
| 掌握学习法 | 布鲁姆教育目标分类 | 每个知识点≥80%正确率才进入下一环节 |
| 间隔重复 | 艾宾浩斯遗忘曲线 | 7次间隔复习：20分钟→当天→次日→第3天→第7天→第14天→第30天 |
| 多感官学习 | 蒙台梭利 | 视觉+听觉+触觉+动觉四位一体 |
| 新加坡CPA教学法 | 新加坡数学 | 具象→图示→抽象，数学全程CPA |
| 认知科学识字法 | 字理识字 | 音-形-义三角联结+语境化识字 |
| 自然拼读（Phonics） | 英语母语国家 | 字母-发音对应规则+拼读训练 |
| TPR全身反应法 | James Asher | 语言+身体动作同步，英语词汇核心方法 |

### 二、从复习到新学的无缝衔接

游戏屋不是简单地按年级从头教，而是采用**诊断先行**的策略：

1. **入学诊断**：15题×3科的"探险任务"，精准定位孩子的知识缺口
2. **起点映射**：不是"从第1课开始"，而是"从孩子不会的第一个地方开始"
3. **小步快赢**：单课时30分钟，I Do仅6分钟，You Do正确率≥80%才过关
4. **潜移默化**：通过I Do→We Do→You Do的流程，学习方法被自然内化，孩子不需要刻意记忆"怎么学"，而是在陪学过程中自然习得

这套机制让小甜这样的孩子，能在不知不觉中把一二三年级落下的知识补回来，同时学会"如何学习"这件事本身。

### 三、知识体系和训练完全符合最新考试方向

课程内容严格对标最新课程标准，每单元均包含"期末真题"或"期中真题"标记的题目，确保练习方向与考试要求一致。具体见 `课程大纲-小学一至三年级语数英补习-最终版.md` 和 `补救教学深度审核与优化方案.md` 的详细设计。

### 四、强制亲子共学机制

这是游戏屋区别于所有教育App的核心设计：**家长必须参与**。

- I Do 阶段：家长照着"家长侧边栏"的话术示范，孩子看
- We Do 阶段：家长和孩子一起做，逐步放手
- You Do 阶段：孩子独立做，家长在旁边观察记录

老田从自己的教训中明白：孩子太小，自控力是靠陪伴建立起来的。把平板扔给孩子自己学，只会重蹈覆辙。

### 五、AI可快速扩展年级和科目

这是游戏屋区别于传统教育软件的架构优势：**所有课程数据都是结构化的TypeScript文件**，新增一个年级或一个科目，只需要让AI按照既定模板生成对应的数据文件即可。当前已覆盖小学1-6年级语数英三科（18个课程包、144个单元、720个课时、5040道练习题），后续可快速扩展至初中，甚至增加科学、编程等新科目。

### 六、三科内容均衡，各有特色

游戏屋的语文、数学、英语三科在内容量上完全对等（各48单元、240课时、1680道练习题），但各有教学特色：

- **语文**：内容最丰富（45万字），以字理识字法为核心，每个生字包含拼音、部首、本义、引申义、组词、儿歌六维详解，配合拼音动画和卡牌揭示
- **数学**：以新加坡CPA教学法为核心，14种动画场景中数学独占8种（数数、合并、分离、竖式计算、凑十法、CPA过渡、比较、图形绘制），让抽象运算可视化
- **英语**：以自然拼读和TPR全身反应法为核心，每课包含情景对话、角色扮演、动作指令，配合颜色混合和场景构建动画

### 七、更多特点

14种GSAP教学动画、**智能动画推断系统**（根据内容自动选择动画场景，不再有纸箱占位符）、**精准网课视频匹配**（每个课时3个免费视频源）、智能TTS语音系统（拼音不再读成英文、emoji 智能聚合）、**库存经济激励系统**（心愿商店+钻石商店支持库存与家长管理）、错题本与知识点溯源、完全离线运行（单文件打包+哈希路由）——详见下方功能介绍。

## 项目亮点

### 完整的课程体系
- **3 学科 × 6 年级 = 18 套课程**，覆盖小学全年级语数英全部知识点
- **144 个单元 · 720 节课时 · 5,040 道练习题**，三科完全对等（各48单元/240课时/1680题）
- 每节课包含：课文内容 + I-Do/We-Do/You-Do 教学闭环 + 7 道分层练习题
- **三科各有特色教学法**：语文（字理识字法）、数学（新加坡CPA教学法）、英语（自然拼读+TPR），根据学科特点量身定制

### 智能动画教学系统
- **14 种动画场景类型**：拼音（语文）、颜色混合（英语）、数数、合并、分离、竖式计算、凑十法、CPA 过渡、图形绘制、卡片揭示、时间线、场景构建、比较等
- **三科各有专属动画**：语文（拼音动画、卡牌揭示）、数学（8种运算可视化动画）、英语（颜色混合、场景构建）
- **GSAP 专业动画引擎**：文字揭示、弹跳出现、打字机、数字滚动、庆祝纸屑等 12 种动画
- **智能动画推断系统**：`inferAnimationConfig` 根据课文内容自动选择最合适的动画场景
  - 9 大英语主题模式匹配：问候、数字、字母、颜色、家庭、动物、水果、身体部位、日常动作
  - **15+ 中文内容模式匹配**：动物、食物、身体部位、家庭、交通、动作、天气、服装、节日、文具、颜色、场所、职业、自然景观、单元回顾、闯关庆祝
  - 中文过程/变化类内容自动识别（变成/循环/步骤 → 时间线动画）
  - 智能场景构建兜底：从内容提取 emoji + 100+ 关键词→emoji 主题映射表，动画效果轮换（bounceIn/scaleIn/slideIn/popIn/fadeIn），**不再出现 📦 纸箱占位符**，改用主题 emoji 或 🎯
- 每个动画可配置 TTS 旁白，视听结合

### 精准网课视频匹配
- **数学一年级真实直链**：`mathGrade1Videos.ts` 包含网易公开课真实视频直链（如"数一数""1-5的加法"等13个课时可直接播放），优先于搜索链接使用
- **动态搜索链接生成**：无直链的课时自动生成 3 个免费视频资源
  - **国家中小学智慧教育平台**（zxx.edu.cn）：教育部官方平台，免费同步课程，内容权威可靠
  - **Bilibili 搜索**：内容丰富，有名师讲解和动画教学视频
  - **网易公开课搜索**：免费系统课程，适合系统学习
- **精准关键词**：以"年级 + 学科 + 课时标题"作为搜索词，确保匹配到对应课程
- **优先级机制**：课程自带视频 > 数学一年级专属直链 > 通用搜索链接，确保有直链时优先使用直链

### 智能语音系统
- **拼音智能转换**：内置完整拼音音节表，`b-a-ba` 自动读成"波-啊-巴"，不再读成英文字母
- **TTS 表情清洗**：智能处理文本中的 emoji，连续重复的 emoji 聚合为"X颗星星""X朵花"等格式，避免 TTS 逐个朗读；正则覆盖 U+2B00-U+2BFF（含 ⭐ 星星），按 Unicode 码点计数，正确处理变体选择符（U+FE0F）
- **统一语速控制**：家长可在设置中统一调节拼音/中文/英文朗读速度（0.5x-1.5x）
- **7 种角色语音**：旁白、老师、学生、小朋友、妈妈、爸爸、系统，不同角色不同语速音调，按性别自动匹配不同人声
- **个性化表扬**：可设置孩子名字，答对题时TTS朗读"小甜你真棒！"等鼓励语
- 基于 Web Speech API，**完全离线**，使用系统自带语音

### 库存经济激励系统
- **双货币体系**：星星（日常奖励）+ 钻石（成就奖励）
- **16 种成就**：初出茅庐、三日连胜、数学小达人、零失误、闪电学习等
- **心愿商店（星星兑换）+ 钻石商店**：用货币兑换真实奖励（冰淇淋、动画片、公园、玩具等）
- **库存系统**：
  - `stock` 字段：`-1`=无限、`0`=售罄、`>0`=可兑换数量
  - 兑换后库存自动递减，售罄商品置灰不可点击
  - 家长凭 PIN 验证后可新增/编辑/补货/删除商品
  - 交易历史（星星/钻石变动记录）持久化到 localStorage
  - 旧数据自动迁移：无 `stock` 字段的旧数据按已购状态补默认库存
- **连续学习打卡**：自动追踪学习天数

### 家长可参与
- **家长中心**：查看三科进度、连续天数、累计星星
- **孩子名字设置**：设置后，答对题目会喊孩子的名字表扬
- **语音速度设置**：统一调节朗读速度，含预设和测试功能
- **进度可视化**：进度条直观展示完成情况
- **家长小贴士**：辅导建议
- **商品管理**：PIN 验证后管理心愿商店与钻石商店商品（增删改补货）

### 完全离线
- 所有课程数据内置（~60,000 行教学内容）
- 语音使用系统 TTS，无需联网
- **单文件打包**（vite-plugin-singlefile），构建产物为单个 HTML 文件，支持 `file://` 协议直接打开
- **哈希路由**（createWebHashHistory）在 `file://` 协议下自动启用，避免刷新 404
- 打包为 Windows 桌面应用，**任何电脑都能用**
- 数据本地存储（localStorage），隐私安全

## 使用 TRAE 开发过程

本项目**全程使用 AI 开发**，从课程体系构建、需求分析、架构设计到代码实现、测试修复，全部借助 AI 完成。

### 开发时间线

1. **需求分析与课程体系构建**：AI完成整个项目的需求分析，包括功能需求、非功能需求、性能需求等。
2. **课程大纲设计**：AI 生成小学 1-3 年级语数英课程大纲，三科完全对等（各 24 单元 / 120 课时 / 840 题），并启动五方多 Agent 交叉审核，确保知识体系准确。
3. **代码实现**：45 个源文件、15 个数据文件、9 个页面、9 个动画组件全部由 TRAE IDE 生成，涵盖 Vue 3 组合式 API、Pinia 状态管理、GSAP 动画等。
4. **多专家交叉验证**：启动数学 / 语文 / 英语三个教育专家 Agent 交叉验证课程数据，修复知识错配，确保教学内容符合最新课程标准。
5. **TTS 拼音修复**：修复拼音被读成英文的严重 Bug，扩展 100+ 拼音映射，支持混合文本智能转换，让 `b-a-ba` 正确读成"波-啊-巴"。
6. **跨平台打包**：Electron 打包 Windows 便携 EXE 与 NSIS 安装包，Capacitor 打包 Android APK，实现一次开发多端部署。
7. **网课视频系统重构**：用动态搜索链接生成取代 360 条易失效的硬编码直链，每课时生成国家智慧教育平台/B站/网易公开课 3 个免费视频源。
8. **智能动画系统重写**：重写 `inferAnimationConfig`，新增 9 大英语主题模式匹配、中文过程类时间线动画、100+ 关键词→emoji 主题映射，消除 📦 纸箱占位符。
9. **库存经济系统**：心愿商店与钻石商店引入 `stock` 库存字段，支持家长 PIN 验证管理商品，交易历史持久化，旧数据自动迁移。
10. **TTS 表情清洗**：扩展 emoji 正则覆盖 U+2B00-U+2BFF（含 ⭐），按 Unicode 码点计数，连续 emoji 聚合为"X颗星星"格式。
11. **4-6年级课程扩展**：新增9个课程数据文件（数学/语文/英语各3个年级），新增2520道练习题、144个课时。经过5轮多Agent交叉验证（数据结构、答案正确性、GSAP动画、内容质量、集成一致性），修复全部发现的问题，TypeScript编译零错误。
