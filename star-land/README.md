## 这个项目的故事

小甜是一个二年级的女孩儿。她的爸爸老田每天忙于工作，早出晚归，渐渐疏忽了对小甜学习的管理。他总觉得"孩子还小，慢慢来"，直到那天——班主任打来电话，请家长到学校谈话。

老田坐在教室里，看着小甜的成绩单，心里像被针扎了一样：孩子的成绩已经滑到了班级倒数。

回家的路上，小甜低着头不说话。老田想发火，却发不出——因为他知道，真正该被责怪的是自己。他太放手了，以为孩子能自己管好自己，却忘了八岁的孩子根本没有这个自控力。

那天晚上，老田做了一个决定：他要亲自陪小甜把落下的功课补回来。可当他翻开课本，发现自己也懵了——凑十法是什么？CPA教学法怎么用？英语单词的发音他也不确定，拼音的声调他也搞不清。

于是，老田想到了用AI快速完成整个教学体系。他把自己对女儿的愧疚、对教育的理解、以及从全球教学法中汲取的精华，全部倾注进了这个程序。他给它起了一个名字：**老甜甜和小甜甜的游戏屋**。

这个程序有一个刻意的设计：**它必须由父母陪着孩子一起用**。老田太清楚，小甜之前的学习之所以滑坡，根源就在于他太放手了。孩子太小，自控力是一点一点培养起来的，不能指望她独自面对屏幕就能学好。所以游戏屋里的每一节课，都有"I Do（家长示范）→ We Do（亲子合练）→ You Do（孩子自练）"三个环节，家长是不可或缺的角色。

老田把他对小甜所有的亏欠，都变成了代码里的一行行温暖：答对题了，程序会喊小甜的名字说"小甜你真棒"；连续学习有星星奖励，攒够了可以兑换去公园玩；每天结束时，父女俩一起记录"今日高光时刻"。

这不是一个冷冰冰的教育App，而是一个父亲写给女儿的道歉信，也是一份承诺：**从今天起，爸爸陪你一起学。**

## 项目最大特点

### 注意
本项目完全是AI完成，我在和小甜一起学习的过程中发现有些练习还是有些问题，特别是填空题，答案和应试教育一样，是标准答案。为此我专门开发了**家长批改**功能：在填空题和闪卡题中，当系统判定孩子答错时，会出现"家长批改"按钮，家长可以查看孩子的答案和参考答案，主观判断是否算孩子答对。这既解决了标准答案过于死板的问题，也逼迫家长必须陪着孩子一起学习——因为只有家长才能判断孩子的答案是否合理。

### 一、学习方法全球最先进

本课程深度融合全球十大教学方法论，并非简单堆砌，而是根据补救教学场景做了系统性整合：

| 教学法 | 来源 | 在游戏屋中的应用 |
|--------|------|-----------------|
| RTI分层干预模型 | 美国特殊教育 | 阶段零精准诊断→分层干预→逐级加码 |
| 显性直接教学（EDI） | 美国直接教学法 | 每课时严格 I Do→We Do→You Do 三步流程 |
| 支架式教学 | 维果茨基最近发展区 | 全支架→半支架→微支架，逐步撤除 |
| 掌握学习法 | 布鲁姆教育目标分类 | 每个知识点≥80%正确率才进入下一环节 |
| 间隔重复 | 艾宾浩斯遗忘曲线 | 7次间隔复习：20分钟→当天→次日→第3天→第7天→第14天→第30天 |
| 多感官学习 | 蒙台梭利 | 视觉+听觉+触觉+动觉四位一体 |
| 新加坡CPA教学法 | 新加坡数学 | 具象→图示→抽象，数学全程CPA |
| 认知科学识字法 | 字理识字 | 音-形-义三角联结+语境化识字 |
| 自然拼读（Phonics） | 英语母语国家 | 字母-发音对应规则+拼读训练 |
| TPR全身反应法 | James Asher | 语言+身体动作同步，英语词汇核心方法 |

### 二、从复习到新学的无缝衔接

游戏屋不是简单地按年级从头教，而是采用**诊断先行**的策略：

1. **入学诊断**：15题×3科的"探险任务"，精准定位孩子的知识缺口
2. **起点映射**：不是"从第1课开始"，而是"从孩子不会的第一个地方开始"
3. **小步快赢**：单课时30分钟，I Do仅6分钟，You Do正确率≥80%才过关
4. **潜移默化**：通过I Do→We Do→You Do的流程，学习方法被自然内化，孩子不需要刻意记忆"怎么学"，而是在陪学过程中自然习得

这套机制让小甜这样的孩子，能在不知不觉中把一二三年级落下的知识补回来，同时学会"如何学习"这件事本身。

### 三、知识体系和训练完全符合最新考试方向

课程内容严格对标最新课程标准，每单元均包含"期末真题"或"期中真题"标记的题目，确保练习方向与考试要求一致。具体见 `课程大纲-小学一至三年级语数英补习-最终版.md` 和 `补救教学深度审核与优化方案.md` 的详细设计。

### 四、强制亲子共学机制

这是游戏屋区别于所有教育App的核心设计：**家长必须参与**。

- I Do 阶段：家长照着"家长侧边栏"的话术示范，孩子看
- We Do 阶段：家长和孩子一起做，逐步放手
- You Do 阶段：孩子独立做，家长在旁边观察记录

老田从自己的教训中明白：孩子太小，自控力是靠陪伴建立起来的。把平板扔给孩子自己学，只会重蹈覆辙。

### 五、AI可快速扩展年级和科目

这是游戏屋区别于传统教育软件的架构优势：**所有课程数据都是结构化的TypeScript文件**，新增一个年级或一个科目，只需要让AI按照既定模板生成对应的数据文件即可。当前已覆盖小学1-3年级语数英三科（9个课程包、72个单元、360个课时、2520道练习题），后续可快速扩展至4-6年级，甚至增加科学、编程等新科目。

### 六、三科内容均衡，各有特色

游戏屋的语文、数学、英语三科在内容量上完全对等（各24单元、120课时、840道练习题），但各有教学特色：

- **语文**：内容最丰富（45万字），以字理识字法为核心，每个生字包含拼音、部首、本义、引申义、组词、儿歌六维详解，配合拼音动画和卡牌揭示
- **数学**：以新加坡CPA教学法为核心，14种动画场景中数学独占8种（数数、合并、分离、竖式计算、凑十法、CPA过渡、比较、图形绘制），让抽象运算可视化
- **英语**：以自然拼读和TPR全身反应法为核心，每课包含情景对话、角色扮演、动作指令，配合颜色混合和场景构建动画

### 七、更多特点

14种GSAP教学动画、智能TTS语音系统（拼音不再读成英文）、游戏化激励、错题本与知识点溯源、完全离线运行——详见下方功能介绍。

## 项目亮点

### 完整的课程体系
- **3 学科 × 3 年级 = 9 套课程**，覆盖小学低年级语数英全部知识点
- **72 个单元 · 360 节课时 · 2,520 道练习题**，三科完全对等（各24单元/120课时/840题）
- 每节课包含：课文内容 + I-Do/We-Do/You-Do 教学闭环 + 7 道分层练习题
- **三科各有特色教学法**：语文（字理识字法）、数学（新加坡CPA教学法）、英语（自然拼读+TPR），根据学科特点量身定制

### 丰富的动画教学
- **14 种动画场景类型**：拼音（语文）、颜色混合（英语）、数数、合并、分离、竖式计算、凑十法、CPA 过渡等
- **三科各有专属动画**：语文（拼音动画、卡牌揭示）、数学（8种运算可视化动画）、英语（颜色混合、场景构建）
- **GSAP 专业动画引擎**：文字揭示、弹跳出现、打字机、数字滚动、庆祝纸屑等 12 种动画
- 每个动画可配置 TTS 旁白，视听结合

### 智能语音系统
- **拼音智能转换**：内置完整拼音音节表，`b-a-ba` 自动读成"波-啊-巴"，不再读成英文字母
- **统一语速控制**：家长可在设置中统一调节拼音/中文/英文朗读速度（0.5x-1.5x）
- **5 种角色语音**：旁白、老师、学生、小朋友、系统，不同角色不同语速音调
- **个性化表扬**：可设置孩子名字，答对题时TTS朗读"小甜你真棒！"等鼓励语
- 基于 Web Speech API，**完全离线**，使用系统自带语音

### 游戏化激励
- **双货币体系**：星星（日常奖励）+ 钻石（成就奖励）
- **16 种成就**：初出茅庐、三日连胜、数学小达人、零失误、闪电学习等
- **心愿商店**：用星星兑换真实奖励（冰淇淋、动画片、公园、玩具），学习→奖励→动力正循环
- **连续学习打卡**：自动追踪学习天数

### 家长可参与
- **家长中心**：查看三科进度、连续天数、累计星星
- **孩子名字设置**：设置后，答对题目会喊孩子的名字表扬
- **语音速度设置**：统一调节朗读速度，含预设和测试功能
- **进度可视化**：进度条直观展示完成情况
- **家长小贴士**：辅导建议

### 完全离线
- 所有课程数据内置（~29,000 行教学内容）
- 语音使用系统 TTS，无需联网
- 打包为 Windows 桌面应用，**任何电脑都能用**
- 数据本地存储，隐私安全

## 快速开始（开发者）

### 环境要求
- Node.js ≥ 18
- npm ≥ 9

### 安装与运行

```bash
# 安装依赖
npm install

# 开发模式（热更新）
npm run dev

# 类型检查
npm run check

# 构建 Web 版
npm run build
```

### 桌面应用打包

```bash
# Windows 便携版（单 exe 文件）
npm run electron:build

# Windows NSIS 安装包（带安装向导）
npm run electron:build:installer
```

打包产物输出到 `release/` 目录。

## 使用 TRAE 开发过程

本项目**全程使用 AI 开发**，从课程体系构建、需求分析、架构设计到代码实现、测试修复，全部借助 AI 完成。

### 开发时间线

1. **需求分析与课程体系构建**：AI完成整个项目的需求分析，包括功能需求、非功能需求、性能需求等。
2. **课程大纲设计**：AI 生成小学 1-3 年级语数英课程大纲，三科完全对等（各 24 单元 / 120 课时 / 840 题），并启动五方多 Agent 交叉审核，确保知识体系准确。
3. **代码实现**：45 个源文件、15 个数据文件、9 个页面、9 个动画组件全部由 TRAE IDE 生成，涵盖 Vue 3 组合式 API、Pinia 状态管理、GSAP 动画等。
4. **多专家交叉验证**：启动数学 / 语文 / 英语三个教育专家 Agent 交叉验证课程数据，修复知识错配，确保教学内容符合最新课程标准。
5. **TTS 拼音修复**：修复拼音被读成英文的严重 Bug，扩展 100+ 拼音映射，支持混合文本智能转换，让 `b-a-ba` 正确读成"波-啊-巴"。
6. **跨平台打包**：Electron 打包 Windows 便携 EXE 与 NSIS 安装包，Capacitor 打包 Android APK，实现一次开发多端部署。

### AI 在项目中的角色

| 环节 | 传统方式 | TRAE AI 方式 |
|------|---------|-------------|
| 课程设计 | 教研团队数月打磨 | AI 生成 + 多 Agent 交叉审核 |
| 代码开发 | 专业前端工程师 | 自然语言描述需求，AI 生成代码 |
| 教学法整合 | 教育学专家研究 | AI 融合全球十大教学法并系统整合 |
| 跨平台部署 | 多端开发团队 | 一套代码，AI 辅助打包多端 |

> 整个项目没有一行代码是手写的——这既是 AI 创造力的证明，也是"陪伴"主题的延伸：AI 帮一个想陪伴女儿却不会写代码的父亲，把爱变成了产品。
