## 一、项目架构速查

### 1.1 技术栈
- **前端框架**: Vue 3 + TypeScript + Vite
- **UI**: Tailwind CSS + Lucide Icons
- **动画**: GSAP
- **状态管理**: Pinia (localStorage持久化)
- **路由**: Vue Router

### 1.2 核心文件清单

| 文件路径 | 用途 | 修改频率 |
|---------|------|---------|
| `star-land/src/types/index.ts` | 全局类型定义 | 低 |
| `star-land/src/data/math/grade1-3.ts` | 数学1-3年级课程数据 | 中 |
| `star-land/src/data/chinese/grade1-3.ts` | 语文1-3年级课程数据 | 中 |
| `star-land/src/data/english/grade1-3.ts` | 英语1-3年级课程数据 | 中 |
| `star-land/src/components/animation/LessonAnimation.vue` | 教学动画核心组件 | 低 |
| `star-land/src/pages/LessonPage.vue` | 课程页面 | 低 |

### 1.3 数据结构约定

```
课程数据层级：
  Subject(学科) → Grade(年级) → Unit(单元) → Lesson(课时) → ContentBlock(内容块) + PracticeQuestion(练习题)

ID命名规范：
  数学: m{年级}u{单元}l{课时}q{题号}  (如 m1u5l1q3)
  语文: c{年级}u{单元}l{课时}q{题号}  (如 c1u2l1q4)
  英语: e{年级}u{单元}l{课时}q{题号}  (如 e3u5l2q7)

课时结构（每单元5课时）：
  l1: 核心知识讲解
  l2: 知识深化/变式
  l3: 应用与拓展
  l4: 形成性评价（闯关测试）
  l5: 综合应用（生活场景+真题）

题目层级（每课时7题）：
  q1-q2: 基础层（直接变式）
  q3-q4: 提高层（一步推理）
  q5-q6: 挑战层（跨知识点综合）
  q7: 回溯层（回溯已学知识点，hint中标注 [知识点]（课程-章节））
```

## 三、修改升级操作流程

### 3.1 标准修改流程

```
1. 读取相关文件 → 深度阅读（R2），禁止只读前50行
2. 理解现有结构 → 追踪执行路径（R3）
3. 制定修改方案 → 单点深入（R5），一次只改一个子系统
4. 实施修改 → 使用Edit精确修改，禁止Write覆盖整个文件
5. 运行验证 → npx vue-tsc --noEmit + npx vite build（R4）
6. 同步文档 → 更新课程大纲/补救方案（R29）
7. 记录日志 → 写入memory日志（R30）
```

## 四、常见陷阱与防范

### 4.1 虚假实现识别

| 陷阱 | 识别方法 | 正确做法 |
|------|---------|---------|
| answer填公式描述 | Grep搜索answer字段中的运算符 | answer必须是计算结果 |
| 题目只有基础层 | 统计每课时题目数<7 | 补充提高+挑战+回溯 |
| 回溯题无来源标注 | Grep搜索q7的hint字段 | 添加[知识点]（课程-章节）格式 |
| 3D图形不可旋转 | 实际运行测试 | 确保mousedown+touchstart绑定 |
| 竖式动画无进位 | 检查verticalCalcConfig.carry | 补充carry数组 |
| 真题标记缺失 | Grep搜索【期末真题】 | 每单元至少1道 |
| 选项歧义 | 人工审查choice题 | 确保答案唯一 |

### 4.2 重复实现防范

- 修改前必须Read当前文件完整内容
- 禁止假设"上次已修改"——必须验证
- 使用Grep确认修改是否已存在
- 同一功能禁止在多处重复实现

### 4.3 错乱防范

- 修改课程数据时，只修改目标文件，不要"顺手"改其他文件
- 新增课时必须追加到单元末尾，不要插入中间
- ID编号必须连续，不要跳号
- 每次修改后立即运行类型检查
